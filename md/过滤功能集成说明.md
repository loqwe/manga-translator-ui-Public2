如果需要暂时禁用过滤，可以：

**方法1**：重命名过滤文件
```bash
mv examples/filter_list.txt examples/filter_list.txt.bak
```

**方法2**：清空过滤规则
编辑 `examples/filter_list.txt`，删除所有规则（保留区域标记）

### 3. 调试过滤规则

如果某些文本被错误过滤或未被过滤：

1. **检查日志**：查找 `过滤文本区域` 关键词
2. **检查OCR结果**：查看 OCR 识别的原文是否准确
3. **测试正则**：使用在线正则测试工具验证
4. **调整规则**：修改 `examples/filter_list.txt` 后重新运行

## 已知问题

### 1. ⚠️ OCR识别错误可能导致过滤失效

**问题**：如果OCR将 "STAFF" 识别为 "STAF" 或 "ST AFF"，过滤规则可能无法匹配。

**解决方案**：
- 使用更宽松的正则表达式：`ST\s*A\s*FF?`
- 添加常见OCR错误的变体

### 2. ⚠️ 大小写敏感

**问题**：虽然正则和包含过滤不区分大小写，但精确过滤会转换为小写比较。

**解决方案**：已自动处理，无需手动调整

## 技术细节

### 正则表达式编译

所有正则表达式在加载时编译并缓存：

```python
# 编译时（启动时执行一次）
pattern = re.compile(r'^>?STAFF$', re.IGNORECASE)
regex_list.append(pattern)

# 匹配时（每个文本块执行一次）
if pattern.search(text):
    return (pattern.pattern, "正则")
```

### 内存占用

- **过滤规则**：~2KB（28条规则）
- **编译后的正则**：~5KB（16个Pattern对象）
- **总内存占用**：< 10KB（可忽略）

- ✅ `load_filter_list()` 使用全局缓存，线程安全
