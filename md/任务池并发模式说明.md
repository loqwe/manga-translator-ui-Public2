# 翻译任务池并发模式

## 背景问题

原有的并发模式存在"一个慢拖全部"的问题：

```
旧模式：批量创建任务 + asyncio.gather 等待
─────────────────────────────────────────
Task A ████████████████████░░░░░░░░░░░░░░  (慢)
Task B ████░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  (快，但要等A)
Task C ██████░░░░░░░░░░░░░░░░░░░░░░░░░░░░  (快，但要等A)
Task D ███░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░░  (快，但要等A)
                                    ▲
                                    └── 全部等最慢的完成
```

## 新架构：任务池模式

```
┌─────────────────────────────────────────────────────────────┐
│                    _translation_worker                       │
├─────────────────────────────────────────────────────────────┤
│                                                             │
│   ┌──────────────┐      ┌─────────────┐                     │
│   │ _batch_      │─────►│ batch_queue │                     │
│   │ collector    │      └──────┬──────┘                     │
│   └──────────────┘             │                            │
│                                ▼                            │
│              ┌─────────────────┼─────────────────┐          │
│              │                 │                 │          │
│         ┌────▼────┐      ┌────▼────┐      ┌────▼────┐      │
│         │Worker-0 │      │Worker-1 │      │Worker-2 │ ...   │
│         └────┬────┘      └────┬────┘      └────┬────┘      │
│              │                 │                 │          │
│              ▼                 ▼                 ▼          │
│         独立处理           独立处理           独立处理        │
│         (互不等待)        (互不等待)        (互不等待)       │
│                                                             │
└─────────────────────────────────────────────────────────────┘
```

### 核心组件

| 组件 | 职责 |
|------|------|
| `_batch_collector` | 从 `translation_queue` 收集图片，凑够批次后放入 `batch_queue` |
| `_translation_pool_worker` | 独立从 `batch_queue` 拉取批次并处理，互不干扰 |
| `_expand_workers_on_inpaint_stop` | 修复线程停止后，动态启动额外 worker |

### 工作流程

1. **启动阶段**（修复线程运行时）
   - 只启动 1 个 worker（串行模式，避免 GPU 资源争用）
   - 收集者持续收集图片、凑批次

2. **扩展阶段**（修复线程停止后）
   - 触发 `inpaint_stopped_event`
   - 启动额外 worker（达到目标并发数）
   - 扩展速率限制器的并发槽位

3. **结束阶段**
   - 收集者完成后，发送 N 个毒丸信号（None）
   - 每个 worker 收到毒丸后优雅退出
   - 等待所有 worker 完成

## 与旧模式对比

```
新模式：任务池持续消费
─────────────────────────────────────────
Worker-0 ████████████████████████████████  (独立处理，不等其他)
Worker-1 ████████████████████████████████  (独立处理，不等其他)
Worker-2 ████████████████████████████████  (独立处理，不等其他)
Worker-3 ████████████████████████████████  (独立处理，不等其他)
```

### 优势

1. **互不干扰**：Worker A 慢了完全不影响 Worker B/C/D
2. **吞吐量最大化**：每个 worker 处理完立即拉取下一个任务
3. **资源可控**：worker 数量 = 并发数，简单明确
4. **动态扩展**：支持运行时增加 worker

## 关键代码位置

- `concurrent_pipeline.py`
  - `_translation_worker()` - 主入口，启动收集者和 worker
  - `_batch_collector()` - 批次收集者
  - `_translation_pool_worker()` - worker 实现
  - `_handle_translation_error()` - 错误处理
  - `_expand_workers_on_inpaint_stop()` - 动态扩展

## 配置参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `batch_size` | 每批次图片数 | 3 |
| `translation_concurrency` | 目标 worker 数 | 1 |
| `preset_rpm` | 每分钟最大请求数 | 0（不限制）|

## 错误处理

- **安全限制错误**：放入延迟重试队列
- **其他错误**：标记失败，不阻塞其他 worker

## 毒丸模式（Poison Pill）

用于优雅关闭 worker：

```python
# 发送毒丸信号
for _ in range(current_workers):
    await batch_queue.put(None)

# Worker 检测毒丸
if batch is None:
    break  # 优雅退出
```
