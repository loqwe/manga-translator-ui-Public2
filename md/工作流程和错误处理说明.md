## 1. 完整工作流程

### 1.1 标准翻译流程

```
用户输入
  ↓
配置加载 (config.json)
  ↓
图片加载 (批量/单张)
  ↓
┌─────────────────────────────────────┐
│ 阶段1: 检测和OCR                     │
│ • 文本检测 (YOLO/DBText)              │
│ • OCR识别 (48px/manga_ocr/paddleocr) │
│ • 文本行合并 (YOLO限制跨气泡合并)     │
│ • 广告过滤 (正则表达式过滤29条规则)   │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 阶段2: 翻译                          │
│ • 上下文构建 (优先译文，无译文用原文) │
│ • 批量翻译 (batch_size控制)          │
│ • 并发控制 (translation_concurrency) │
│ • 速率限制 (translation_rpm)         │
│ • 术语提取 (extract_glossary)        │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 阶段3: 修复                          │
│ • 蒙版细化 (mask_refinement)         │
│ • 图片修复 (lama_large/sd)           │
└─────────────────────────────────────┘
  ↓
┌─────────────────────────────────────┐
│ 阶段4: 渲染                          │
│ • 文字渲染 (字体、布局、断行)        │
│ • 图片合成 (alpha通道处理)           │
│ • 文件保存 (图片+JSON)               │
└─────────────────────────────────────┘
  ↓
输出结果
```

### 1.2 关键配置参数

#### 核心参数

```json
{
  "cli": {
    "context_size": 3,              // 上下文页数 (0=禁用)
    "batch_size": 3,                // 翻译批次大小
    "batch_concurrent": true,       // 并发流水线模式
    "translation_concurrency": 4,   // 翻译并发数
    "translation_rpm": 10,          // 速率限制 (RPM)
    "use_gpu": true,                // GPU加速
    "save_quality": 100             // 输出质量
  },
  "translator": {
    "translator": "openai_hq",      // 翻译器选择
    "target_lang": "CHS",           // 目标语言
    "extract_glossary": true        // 术语提取
  }
}
```

### 1.3 特殊处理模式

#### 不支持并发流水线的模式

| 模式 | 说明 | 原因 |
|------|------|------|
| `load_text` | 从JSON加载翻译 | 无需翻译，直接渲染 |
| `template + save_text` | 导出原文 | 需要顺序保存 |
| `generate_and_export` | 导出翻译 | 需要顺序保存 |
| `colorize_only` | 仅上色 | 无需翻译/修复 |
| `upscale_only` | 仅超分 | 无需翻译/修复 |
| `inpaint_only` | 仅修复 | 无需翻译 |

当启用这些模式时，系统会自动退回到顺序处理模式。

## 2. 并发流水线处理

### 2.1 流水线架构

#### 简化架构图

```
┌──────────────────────────────────────────────────────┐
│               并发流水线 (4线程并发)                   │
├──────────────────────────────────────────────────────┤
│                                                        │
│  [检测+OCR线程] ──→ 分批加载图片                       │
│       ↓                                                │
│  ┌───────────────┐                                    │
│  │ 翻译队列      │ ←── [翻译线程] 批量并发翻译          │
│  │ 修复队列      │ ←── [修复线程] GPU修夏              │
│  └───────────────┘                                    │
│       ↓           ↓                                    │
│  [渲染线程] ←─── 合并结果 + 保存                       │
│                                                        │
└──────────────────────────────────────────────────────┘
```

#### 详细执行流程

```
输入: 10张图片 | batch_size=3 | concurrency=4

时间轴  线程1:检测+OCR    线程2:翻译      线程3:修复      线程4:渲染
────────────────────────────────────────────────────────────
0-2s   加载001.jpg
       检测+OCR
       →翻译队列
       →修复队列

2-4s   加载002.jpg      等待批次     等待翻译     等待修复
       检测+OCR
       →队列(1/3)

4-6s   加载003.jpg      等待批次     等待翻译     等待修复
       检测+OCR
       →队列(2/3)

6-8s   加载004.jpg      批次已满!      001修复中     等待修复
       检测+OCR      翻译001-003
       →队列(3/3)   (4个并发)

8-10s  加载005.jpg      翻译中...     002修复中     001渲染中
       检测+OCR                                  →保存

10-12s 加载006.jpg      翻译完成!     003修复中     002渲染中
       检测+OCR      收集004-006                  →保存
                       (1/3)

12-14s 加载007.jpg      收集中...     004修复中     003渲染中
       检测+OCR      (2/3)                      →保存

14-16s 加载008.jpg      批次已满!     005修复中     004渲染中
       检测+OCR      翻译004-006                  →保存

16-18s 加载009.jpg      翻译中...     006修复中     005渲染中
       检测+OCR                                  →保存

18-20s 加载010.jpg      翻译完成!     007修复中     006渲染中
       检测+OCR      收集007-009                  →保存
       ↓完成

20-22s 闲置           批次已满!     008修复中     007渲染中
                       翻译007-009                  →保存

22-24s 闲置           翻译中...     009修复中     008渲染中
                                                      →保存

24-26s 闲置           翻译完成!     010修复中     009渲染中
                       收集010                       →保存
                       (1/3)

26-28s 闲置           翻译010       修复完成!     010渲染中
                                                      →保存

28-30s 闲置           完成          闲置          完成!
────────────────────────────────────────────────────────────
总耗时: 30秒 (平均 3秒/张)

对比顺序处理: 约 80秒 (平均 8秒/张)
效率提升: 2.7倍
```

### 2.2 动态并发调整

#### 调整规则

```python
# 规则1: 修复队列堆积 → 降低翻译并发
if inpaint_queue_size > 3 and current_concurrency > 1:
    current_concurrency = 1
    set_translation_concurrency(1)  # 术语直接写入模式
    logger.info("⚠️ [翻译] 修复队列未清空 (X 张)，降低翻译并发数: 4 -> 1")

# 规则2: 修复队列清空 → 恢复翻译并发
if inpaint_queue_size == 0 and concurrency_adjusted:
    current_concurrency = original_concurrency
    set_translation_concurrency(original_concurrency)  # 术语缓存模式
    logger.info("✅ [翻译] 修复队列已清空，恢复翻译并发数: 1 -> 4")
```

#### 监控日志

```log
[翻译] 收集到图片 (2/3)
[翻译] 批次已满 (3/3)，开始翻译
[翻译] 批量翻译 3 张图片
⚠️ [翻译] 修复队列未清空 (5 张)，降低翻译并发数: 4 -> 1
[修复] 完成: 001.jpg (5/10)
✅ [翻译] 修复队列已清空，恢复翻译并发数: 1 -> 4
```

### 2.3 术语缓存策略

#### 写入策略切换

| 并发数 | 策略 | 说明 |
|--------|------|------|
| = 1 | 直接写入 | 每个术语立即写入 `prompt.json` |
| > 1 | 缓存批量写入 | 缓存到内存，翻译完成后一次性写入 |

```

## 3. 常见错误及解决方案

### 3.1 API相关错误

#### 错误1: 速率限制 (Rate Limit)

**错误信息**:
```
Error code: 429 - Rate limit exceeded
```

**原因**:
- 翻译并发数过高
- RPM设置超过API限制
- 短时间内请求过多

**解决方案**:
```json
// 降低并发参数
{
  "cli": {
    "translation_concurrency": 1,  // 降为1
    "translation_rpm": 10          // 降低RPM
  }
}
```

**日志验证**:
```log
[翻译] 当前并发任务数: 1
[翻译] 批量翻译 1 张图片
```

---

#### 错误2: 内容安全过滤 (Safety Filter)

**错误信息**:
```
API error: Content filtered due to safety policy
The response was blocked due to content filtering
```

**原因**:
- AI检测到敏感内容（R18+、暴力等）
- 即使有破限词，部分内容仍可能被过滤

**解决方案**:

1. **检查破限词配置**:
```bash
# 确认破限词存在
cat dict/system_prompt_hq.json | grep "NO CENSORSHIP"
```

2. **使用更宽松的翻译器**:
- OpenAI HQ (gpt4o-hq) - 有破限词
- Gemini HQ (gemini-hq) - 有破限词
- 避免使用基础版翻译器

3. **批量翻译时自动拆分重试**:
系统会自动检测安全过滤错误并拆分为单张重试：

```log
[翻译] 批量翻译触发安全限制，拆分为 3 个单张批次重试
[翻译] 单张重试 1/3: 001.jpg
[翻译] 单张重试成功: 001.jpg
```

---

#### 错误3: 上下文过长 (Context Length)

**错误信息**:
```
This model's maximum context length is 128000 tokens
```

**原因**:
- 上下文页数过多
- 图片文本过多
- 累积的上下文超过模型限制

**解决方案**:
```json
{
  "cli": {
    "context_size": 2,  // 减少上下文页数 (从3降到2)
    "batch_size": 2     // 减少批次大小
  }
}
```

**日志验证**:
```log
[上下文] 已启用上下文翻译，历史页数: 2
[上下文] 携带 2 页上下文，15 个句子作为翻译参考
```

---

### 3.2 并发流水线错误

#### 错误4: 修复队列堆积

**症状**:
```log
⚠️ [翻译] 修复队列未清空 (8 张)，降低翻译并发数: 4 -> 1
```

**原因**:
- GPU性能不足
- 修复速度 < 翻译速度
- 内存占用过高

**解决方案**:

1. **降低翻译并发数**:
```json
{
  "cli": {
    "translation_concurrency": 2  // 从4降到2
  }
}
```

2. **提升GPU性能**:
```json
{
  "inpainter": {
    "inpainting_size": 1024,      // 从2048降到1024
    "inpainting_precision": "fp16" // 使用半精度
  }
}
```

3. **系统会自动调整**:
系统检测到堆积会自动降为1，无需手动干预。

---

#### 错误5: 基础上下文丢失

**错误信息**:
```log
[翻译] 找不到 001.jpg 的基础上下文
[修复] 找不到 001.jpg 的基础上下文
```

**原因**:
- 检测+OCR阶段出错
- ctx对象未正确保存到 `base_contexts`
- 并发流水线内存管理问题

**解决方案**:

1. **检查检测+OCR日志**:
```log
[检测+OCR] 失败: 错误信息
```

2. **临时禁用并发模式**:
```json
{
  "cli": {
    "batch_concurrent": false  // 回退到顺序模式
  }
}
```

3. **检查图片文件**:
- 文件是否损坏
- 路径是否包含特殊字符
- 文件权限是否正确

---

#### 错误6: 数量不匹配 (Translation Count Mismatch)

**错误信息**:
```log
[翻译] 警告: 翻译结果数量不匹配
输入: 12个文本块, 输出: 10个翻译
ValueError: Translation count mismatch
```

**原因**:
- AI返回的翻译数量与输入不一致
- JSON解析错误
- API截断响应
- 安全过滤导致部分结果丢失

**解决方案**:

1. **系统自动处理**:
系统会自动补齐缺失的翻译：
```python
# 自动补齐逻辑
if len(translations) < len(input_texts):
    # 用原文填充缺失的翻译
    for i in range(len(translations), len(input_texts)):
        translations.append(input_texts[i])
```

2. **减小批次大小**:
```json
{
  "cli": {
    "batch_size": 2  // 从3降到2
  }
}
```

3. **检查API响应**:
- 查看API是否有错误信息
- 检查是否触发token限制
- 尝试使用更小的上下文

**日志示例**:
```log
⚠️ [翻译] 翻译数量不匹配: 预期 12, 实际 10
[翻译] 自动填充缺失的 2 个翻译
✅ [翻译] 已修复数量不匹配问题
```

#### 错芆12: JSON解析错误

**错误信息**:
```log
JSONDecodeError: Expecting property name enclosed in double quotes
ValueError: Invalid JSON response from API
```

**原因**:
- AI返回的响应不是标准JSON格式
- 包含 Markdown 格式 (```json...```)
- 翻译内容包含未转义的引号
- API响应被截断

**解决方案**:

1. **系统自动清理**:
```python
# 自动移除 Markdown 标记
if response.startswith('```'):
    response = response.split('```json')[1].split('```')[0]
```

2. **重试机制**:
系统会自动重试1-2次：
```log
[翻译] JSON解析失败，第1次重试...
[翻译] 重试成功
```

3. **检查提示词**:
确保系统提示词要求JSON输出：
```json
{
  "system_prompt": "...Your output MUST be a valid JSON object..."
}
```

**日志示例**:
```log
⚠️ [翻译] JSON解析失败: Expecting property name
[翻译] 原始响应: ```json\n{"translations": [...]}\n```
[翻译] 清理后的响应: {"translations": [...]}
✅ [翻译] JSON解析成功
```
