# 禁翻表功能说明

## 功能概述

禁翻表功能用于识别和处理漫画中的广告内容，例如赞助商信息、社交媒体链接、捐赠请求等。当检测到广告内容时，系统会根据配置的处理模式进行相应操作，避免不必要的翻译或保留原文。

## 配置选项

在配置文件中，可以通过以下选项控制禁翻表功能：

```python
# 启用广告内容过滤（禁翻表）
enable_ad_filter: bool = False

# 广告内容关键词列表，用于识别广告文本
ad_keywords: List[str] = [
    "paypal", "discord", "telegram", "donate", "donation", "support", 
    "patreon", "sponsor", "https://", "http://", "www.", ".com", ".me", ".gg"
]

# 广告处理模式
# 'text'=保留原文不翻译，'blank'=清空文本，'skip'=完全跳过该区域
ad_skip_mode: str = "text"
```

## 处理模式说明

禁翻表功能提供三种处理模式：

1. **文本模式 (text)**：保留广告内容的原文，不进行翻译。这适用于需要保留原始链接或信息的情况。

2. **空白模式 (blank)**：将广告内容清空，在最终输出中显示为空白区域。这适用于希望完全移除广告内容但保留其位置的情况。

3. **跳过模式 (skip)**：完全跳过广告区域，不进行任何处理。这适用于希望在最终输出中完全忽略广告区域的情况。

## 工作原理

1. 在翻译过程中，系统会检查每个文本区域是否包含广告关键词。
2. 如果检测到广告内容，系统会根据配置的处理模式进行处理。
3. 对于非广告内容，系统会正常进行翻译处理。

## 自定义广告关键词

您可以根据需要自定义广告关键词列表，以便更准确地识别特定类型的广告内容。关键词匹配是不区分大小写的，系统会将文本和关键词都转换为小写进行比较。

## 示例

假设有以下文本区域：

```
Support us on Patreon: https://www.patreon.com/example
```

在不同处理模式下的结果：

- **文本模式 (text)**：保留原文 "Support us on Patreon: https://www.patreon.com/example"
- **空白模式 (blank)**：清空文本，显示为空白区域
- **跳过模式 (skip)**：完全跳过该区域，不进行任何处理

## 注意事项

1. 禁翻表功能默认是禁用的，需要在配置中手动启用。
2. 广告关键词匹配是基于简单的字符串包含关系，可能会有误判。
3. 如果您发现某些非广告内容被误判为广告，可以调整关键词列表或暂时禁用此功能。

## 技术实现

禁翻表功能通过 `AdFilter` 类实现，主要包含以下方法：

- `is_ad_content(text, config)`：检查文本是否包含广告内容
- `handle_ad_content(text, config)`：根据配置处理广告内容
- `process_translations(texts, translations, config)`：处理翻译结果，应用广告内容处理
