# 漫画翻译完整流程说明

基于代码分析，本文档描述从导入图片到导出翻译结果的完整流程。

## 一、整体流程概览

```
用户选择图片
     ↓
1. 图片预处理（上色、超分）
     ↓
2. 文字检测 (Detection)
     ↓
3. OCR 识别原文
     ↓
4. 文本区域合并 (Textline Merge)
     ↓
5. 加载提示词和术语 ← 作品识别
     ↓
6. 翻译 (Translation)
     ↓
7. 术语自动提取（可选）
     ↓
8. 后处理（标点修正、后置字典）
     ↓
9. 修复涂白 (Inpainting)
     ↓
10. 渲染译文 (Rendering)
     ↓
11. 保存输出图片 + translation_map.json
```

## 二、详细流程

### 2.1 图片路径存储

```python
# 入口：translate() 方法
image.name = "D:\漫画\飞机杯女神连线中\第01话\001.jpg"

# 存入 Context
ctx.image_name = image.name
```

### 2.2 作品识别流程

**代码位置**: `manga_translator.py` → `_load_and_prepare_prompts()`

```python
# 1. 获取图片路径
image_name = getattr(ctx, 'image_name', None)

# 2. 解析作品名
inferred_work = resolve_work_name(image_name)
```

**`resolve_work_name()` 内部逻辑** (`work_resolver.py`):

```
D:\output\result\001.jpg (翻译后图片)
     ↓
1. get_source_path_from_translation_map()
   └─ 查找 translation_map.json，追溯原图路径
   └─ → "D:\漫画\飞机杯女神连线中\第01话\001.jpg"
     ↓
2. infer_raw_work_name_from_image_path()
   └─ 跳过无效目录: result, output, temp...
   └─ 跳过章节目录: 第01话, Chapter 1, 001...
   └─ → "飞机杯女神连线中"
     ↓
3. map_translated_work_name() 
   └─ 查询 name_mapping.json
   └─ 生肉名 → 熟肉名
```

### 2.3 语言提示词加载

**代码位置**: `openai_hq.py` → `_build_system_prompt()`

```python
# 根据源语言选择提示词文件
lang_to_prompt_file = {
    "JPN": "ja.json",   # 日语
    "KOR": "ko.json",   # 韩语  
    "ENG": "en.json",   # 英语
    "IND": "id.json",   # 印尼语
    "ESP": "es.json",   # 西班牙语

prompt_filename = lang_to_prompt_file.get(source_lang, "default.json")
prompt_path = os.path.join(BASE_PATH, 'dict', 'prompts', prompt_filename)
```

**提示词文件结构** (`dict/prompts/ko.json`):
```json
{
  "system_prompt": "角色设定：深耕ACG圈十余年的资深汉化组翻译主笔...\n核心约束：强制粗俗露骨风格、保留语气、拟声词处理...\n输出格式：JSON对象..."
}
```

### 2.4 术语加载流程

**代码位置**: `manga_translator.py` → `_load_and_prepare_prompts()`

```python
# 1. 加载提示词文件（包含术语）
with open(prompt_path, 'r', encoding='utf-8') as f:
    ctx.custom_prompt_json = json.load(f)

# 2. 剔除 unassigned（未确定作品名的术语不参与翻译）
glossary_obj.pop("unassigned", None)

# 3. 按作品筛选术语
filtered_prompt, selected_work = filter_custom_prompt_json_by_work(
    ctx.custom_prompt_json,
    inferred_work
)
```

**术语筛选规则** (`filter_custom_prompt_json_by_work`):

| 情况 | 行为 |
|------|------|
| 能识别作品且存在于 works 中 | 返回该作品的术语 |
| 无法识别作品，但有 raw_work_name | 从 unassigned 中按 raw_work_name 匹配术语 |
| 无法识别作品，无 raw_work_name | 返回空术语表 |

**术语文件结构** (`dict/prompt_example.json`):
```json
{
  "system_prompt": "可选的额外系统提示...",
  "glossary": {
    "works": {
      "飞机杯女神连线中": {
        "Person": [{"original": "교주", "translation": "教主", "lang": "ko"}],
        "Location": [...],
        ...
      }
    },
    "unassigned": {
      "Person": [
        {"original": "교주", "translation": "教主", "lang": "ko", "raw_work_name": "飛行機カップ女神オンライン中"}
      ]  // 未确定作品名，可通过 raw_work_name 匹配
    }
  }
}
```

### 2.5 最终提示词组合

**代码位置**: `openai_hq.py` → `_build_system_prompt()`

```python
final_prompt = ""

# 1. AI断句提示词（如果启用）
if line_break_prompt_str:
    final_prompt += f"{line_break_prompt_str}\n\n---\n\n"

# 2. 用户自定义提示词 + 术语（筛选后）
if custom_prompt_str:
    final_prompt += f"{custom_prompt_str}\n\n---\n\n"

# 3. 语言专用基础提示词
final_prompt += base_prompt

# 4. 术语提取提示词（如果启用）
if extract_glossary:
    final_prompt += f"\n\n---\n\n{extraction_prompt}"
```

### 2.6 HQ 批量翻译上下文传递

**代码位置**: `manga_translator.py` → `translate_batch_high_quality()`

当使用高质量翻译器（openai_hq/gemini_hq）进行批量翻译时，系统会构建 `enhanced_ctx` 作为合并上下文：

```python
# 创建合并上下文
enhanced_ctx = Context()

# ✅ 复制第一张图片的必要属性
if preprocessed_contexts:
    first_ctx = preprocessed_contexts[0][0]
    enhanced_ctx.input = first_ctx.input      # 图片数据
    enhanced_ctx.from_lang = first_ctx.from_lang  # 源语言
    enhanced_ctx.image_name = first_ctx.image_name  # ★ 作品名识别关键字段

# 合并所有页面的 text_regions
enhanced_ctx.text_regions = all_regions
enhanced_ctx.high_quality_batch_data = batch_data
```

**`image_name` 传递的重要性**：

| 场景 | image_name 来源 | 作品名识别结果 |
|------|-----------------|------------------|
| 第一次批量翻译 | 每张图片的文件路径 | 正常识别 |
| 延迟重试队列重试 | 保存在 ctx 中的原始路径 | 正常识别 |
| 空内容降级单图重试 | 从 batch_contexts 复制 | 正常识别 |
| 重启后重试（旧版） | 可能缺失 | 显示"未匹配到作品" |

**修复后的行为**：
- HQ 批量翻译时，`enhanced_ctx.image_name` 从第一张图片复制
- 确保术语加载时能正确识别作品名
- 延迟重试和降级时保持上下文完整

### 2.7 翻译执行

**代码位置**: `manga_translator.py` → `_run_text_translation()`

```python
# 调用翻译器
translated_sentences = await self._dispatch_with_context(config, texts, ctx)

# 应用翻译结果
for region, translation in zip(ctx.text_regions, translated_sentences):
    region.translation = translation
```

### 2.8 术语自动提取与写回

**代码位置**: `openai_hq.py` → `_translate_batch_high_quality()`

```python
# 解析 AI 响应，提取翻译和新术语
translations, new_terms = parse_hq_response(result_text)

# 术语写回
if extract_glossary and new_terms:
    # 标记所属作品名
    work_name = getattr(ctx, 'glossary_write_work_name', None)
    raw_work_name = getattr(ctx, 'raw_work_name', None)
    for t in new_terms:
        if work_name:
            t['work_name'] = work_name  # 写入 works 区
        elif raw_work_name:
            t['raw_work_name'] = raw_work_name  # 写入 unassigned + 生肉名
    
    # 写入文件
    add_glossary_terms(prompt_path, new_terms)
```

**术语写回规则**:

| 情况 | 写入位置 | 字段 |
|------|----------|------|
| 能识别作品 (`glossary_write_work_name` 有值) | `glossary.works[作品名]` | original, translation, lang |
| 无法识别作品，有生肉名 | `glossary.unassigned` | original, translation, lang, **raw_work_name** |
| 无法识别作品，无生肉名 | `glossary.unassigned` | original, translation, lang |

## 五、常见问题

### Q1: 为什么术语没有生效？
- 检查作品名是否能被正确识别（看日志中的 `ctx.work_name`）
- 检查术语是否在 `glossary.works[作品名]` 中
- `unassigned` 中的术语不会被应用

### Q2: 为什么新提取的术语写到了 unassigned？
- 无法从图片路径识别作品名
- 检查目录结构是否符合 `作品名/章节/图片.jpg` 格式
- 检查 `name_mapping.json` 是否有对应映射
- **注意**: unassigned 中的术语现在会带有 `raw_work_name` 字段，可通过该字段匹配

### Q4: unassigned 中的术语如何生效？
- 当无法识别作品名时，系统会从图片路径提取原始生肉名 (`raw_work_name`)
- 然后从 `unassigned` 中查找 `raw_work_name` 匹配的术语
- 这样即使没有配置 `name_mapping.json`，也能读取到相关术语
- 在术语管理界面的"未确定作品名"标签页可查看/编辑生肉名

### Q3: 翻译后的图片如何追溯原图？
- 查看同目录的 `translation_map.json`
- 格式: `{翻译后路径: 原图路径}`

### Q5: HQ 翻译重试时为什么显示"未匹配到作品"？
- **原因**：早期版本的 HQ 批量翻译未正确传递 `image_name` 到合并上下文
- **影响**：术语加载时无法识别作品，因此使用默认术语或不使用术语
- **修复**：现已在 `enhanced_ctx` 构建时复制第一张图片的 `image_name`
- **日志位置**：查找 `匹配作品` 或 `未匹配到作品` 关键词

### Q6: HQ 翻译返回空内容时怎么处理？
- 系统会自动触发三级降级策略：
  1. HQ 批量翻译失败 → 拆分为单图重试
  2. HQ 单图翻译失败 → 降级到 openai 文本翻译
  3. 文本翻译失败 → 返回原文（不中断流程）
- 日志关键词: `[HQ降级]`, `[降级翻译]`
