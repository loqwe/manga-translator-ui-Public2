# 翻译器重试机制完全指南

## 1. 重试机制概述

### 1.1 重试机制的目的
翻译器重试机制旨在解决OpenAI API调用过程中可能遇到的各种错误和异常情况，提高翻译成功率和质量。主要解决以下问题：
- 内容安全过滤导致的翻译失败
- 翻译结果行数与原文行数不匹配
- API调用超时或网络错误
- 翻译质量不达标

### 1.2 重试机制的核心组件
1. **全局重试次数控制**：通过`attempts`参数控制最大重试次数
2. **多级降级策略**：从批量处理逐步降级到单张处理和纯文本模式
3. **同一对话重试**：在同一对话中添加纠正指令进行重试
4. **精确逐行翻译模拟器**：专门解决翻译结果行数不匹配问题

## 2. 重试次数配置详解

### 2.1 全局重试次数配置
```python
# 在CommonTranslator类中定义
self.attempts = -1  # 默认值为-1，表示无限重试
```

- **配置来源**：从配置文件中读取，通过`parse_args`方法设置
```python
def parse_args(self, config):
    self.attempts = getattr(config, 'attempts', self.attempts)
```

- **同步机制**：设置`attempts`后会同步到`_max_total_attempts`
```python
# 同步 attempts 到 _max_total_attempts
self._max_total_attempts = self.attempts
```

- **实际影响**：从运行日志可以看出，当`attempts`设置为1时，会严格限制重试次数：
```
2025-12-18 19:52:53,966 - WARNING - [manga-translator.OpenAIHighQualityTranslator] - Reached max total attempts: 1/1
2025-12-18 19:52:53,966 - ERROR - [manga-translator.OpenAIHighQualityTranslator] - Reached global attempt limit. Stopping translation.
2025-12-18 19:52:53,967 - WARNING - [manga-translator.OpenAIHighQualityTranslator] - [HQ翻译] 批量翻译失败 1/2: 达到最大尝试次数 (1)，最后一次错误: Unknown error
```

### 2.2 重试次数的使用方式
- **主循环控制**：
```python
max_retries = self.attempts
attempt = 0
is_infinite = max_retries == -1
while is_infinite or attempt < max_retries:
    # 重试逻辑
    attempt += 1
```

- **全局尝试计数**：
```python
# 检查全局尝试次数
if not self._increment_global_attempt():
    raise Exception(f"达到最大尝试次数 ({self._max_total_attempts})，最后一次错误: {last_error_msg}")
```

### 2.3 重试次数的特殊处理
- **无限重试模式**：当`attempts = -1`时，启用无限重试模式
```python
is_infinite = max_retries == -1
if is_infinite:
    # 无限重试逻辑
```

- **单张重试固定次数**：批量拆分为单张后，固定尝试1次
```python
self._reset_global_attempt_count()
self._max_total_attempts = 1
```

- **纯文本回退固定次数**：纯文本翻译模式固定尝试1次
```python
text_translator = OpenAITranslator()
text_translator.attempts = 1  # 纯文本只尝试1次
```

## 3. 安全限制重试流程详解

### 3.1 安全限制错误识别
```python
error_msg = str(batch_error).lower()
is_safety_error = 'content_filter' in error_msg or 'safety' in error_msg
```

### 3.2 三级降级策略完整流程

#### 3.2.1 第一级：高质量批量翻译
- **尝试次数**：使用配置的`attempts`值（默认为-1，无限重试）
- **处理方式**：一次处理多张图片的所有文本
- **错误处理**：检测到安全限制错误时，降级到第二级

#### 3.2.2 第二级：高质量单张翻译
- **触发条件**：
```python
if (is_safety_error or is_mismatch_error) and len(batch_data) > 1:
    self.logger.warning(f"[HQ翻译] 批量翻译失败（{'安全限制' if is_safety_error else '数量不匹配'}），分割为 {len(batch_data)} 个单张批次重试...")
```

- **尝试次数**：固定为1次
```python
self._reset_global_attempt_count()
self._max_total_attempts = 1
```

- **处理流程**：
```python
for img_idx, single_data in enumerate(batch_data):
    # 提取单张图片的文本
    img_texts = single_data.get('texts', [])
    if not img_texts:
        text_regions = single_data.get('text_regions', [])
        img_text_count = len(text_regions)
    else:
        img_text_count = len(img_texts)
    
    # 单张重试（1次）
    try:
        self.logger.info(f"[HQ翻译] 单张重试 {img_idx+1}/{len(batch_data)}: {img_text_count}条文本")
        single_translations = await self._translate_batch_high_quality(
            texts=img_queries,
            batch_data=[single_data],
            # 其他参数...
        )
    except Exception as single_error:
        # 错误处理，可能降级到第三级
```

#### 3.2.3 第三级：纯文本翻译回退
- **触发条件**：
```python
single_error_msg = str(single_error).lower()
single_is_safety = 'content_filter' in single_error_msg or 'safety' in single_error_msg

if single_is_safety:
    self.logger.warning(f"[HQ翻译] 单张重试因安全限制失败，尝试纯文本翻译...")
```

- **尝试次数**：固定为1次
```python
text_translator = OpenAITranslator()
text_translator.attempts = 1  # 纯文本只尝试1次
```

- **处理流程**：
```python
try:
    text_translator = OpenAITranslator()
    text_translator.api_key = self.api_key
    text_translator.base_url = self.base_url
    text_translator.model = self.model
    text_translator.attempts = 1
    
    text_translations = await text_translator._translate(
        from_lang, to_lang, img_queries, ctx
    )
    all_translations.extend(text_translations)
    self.logger.info(f"[HQ翻译] 纯文本翻译成功 {img_idx+1}/{len(batch_data)}")
except Exception as text_error:
    self.logger.error(f"[HQ翻译] 纯文本翻译也失败: {text_error}")
    self.logger.error(f"[HQ翻译] 跳过图片 {img_idx+1}，停止翻译进程")
    raise Exception(f"安全限制：图片 {img_idx+1} 翻译失败（含纯文本回退），停止进程") from text_error
```

### 3.3 单张图片安全限制错误特殊处理
当批次本身就只有一张图片时，直接尝试纯文本翻译：
```python
elif is_safety_error and len(batch_data) == 1:
    self.logger.warning(f"[HQ翻译] 单张图片因安全限制失败，尝试纯文本翻译...")
    try:
        text_translator = OpenAITranslator()
        text_translator.attempts = 1
        # 纯文本翻译逻辑
    except Exception as text_error:
        # 错误处理
```

### 3.4 HQ 空内容错误特殊处理
当 openai_hq 返回 HTTP 200 但内容为空时，触发专门的降级流程：

**代码位置**: `manga_translator.py` → `_batch_translate_texts()`

**触发条件**：
```python
def _is_openai_hq_empty_content_error(err: Exception) -> bool:
    msg = str(err) or ""
    low = msg.lower()
    return (
        "returned empty content" in low
        or "api returned empty text" in low
        or "返回空内容" in msg
        or "返回空文本" in msg
    )
```

**处理流程**：
```
openai_hq 批量翻译返回空内容
    ↓
从 ctx.high_quality_batch_data 获取批次数据
    ↓
按图片拆分为单图任务
    ↓
对每张图片：
  ├── 构建 single_ctx（复制 image_name、from_lang、术语等）
  ├── 调用 HQ 单图翻译
  └── 若失败，调用 _fallback_batch_text_translate()
    ↓
合并结果返回
```

**与安全限制的区别**：
- 安全限制在 HQ 翻译器内部处理（`openai_hq.py`）
- 空内容降级在批量翻译入口处理（`manga_translator.py`）
- 空内容失败后返回原文，不中断整体流程

从日志中可以看出，当批量翻译失败时，系统会自动降级到纯文本翻译模式：
```
2025-12-18 19:52:54,151 - WARNING - [manga_translator] - [HQ翻译] 批量翻译失败: 达到最大尝试次数 (1)，最后一次错误: Unknown error
2025-12-18 19:52:54,151 - INFO - [manga_translator] - [HQ翻译] 降级到文本翻译模式...
2025-12-18 19:52:54,241 - INFO - [manga-translator.OpenAITranslator] - 使用OpenAI纯文本翻译模式处理1个文本，最大尝试次数: 2
```

注意到纯文本翻译模式的最大尝试次数被设置为2，而不是代码中预期的1次。这可能是因为配置文件中的设置或其他代码逻辑影响了这个值。

## 4. 数量不匹配重试流程详解

### 4.1 数量不匹配错误识别
```python
is_mismatch_error = 'mismatch' in error_msg or '不匹配' in error_msg
```

### 4.2 两种处理策略详解

#### 4.2.1 批次拆分策略
- **触发条件**：与安全限制共用相同的拆分逻辑
```python
if (is_safety_error or is_mismatch_error) and len(batch_data) > 1:
```

- **处理流程**：与安全限制重试流程相同，将批次分割成单张图片单独处理
- **单张失败处理**：
```python
# 数量不匹配：单张也失败，跳过图片，停止进程
self.logger.error(f"[HQ翻译] 单张重试失败 {img_idx+1}/{len(batch_data)}: {single_error}")
self.logger.error(f"[HQ翻译] 跳过该图片，停止翻译进程")
raise Exception(f"翻译数量不匹配：图片 {img_idx+1} 翻译失败，停止进程") from single_error
```

#### 4.2.2 精确逐行翻译模拟器
- **触发条件**：
```python
if len(translations) != len(texts):
    retry_attempt += 1
    retry_reason = f"Translation count mismatch: expected {len(texts)}, got {len(translations)}"
```

- **重试次数控制**：
```python
if not is_infinite and attempt >= max_retries:
    raise Exception(f"Translation count mismatch after {max_retries} attempts: expected {len(texts)}, got {len(translations)}")
```

- **同一对话重试机制**：
```python
# 使用同一对话继续重试：追加assistant回复和纠正指令
messages.append({"role": "assistant", "content": result_text})
```

- **精确指令构建**：
```python
texts_display = '\n'.join(texts)
correction_msg = f"""错误：先前输出包含 {len(translations)} 行译文，但原文共有 {len(texts)} 行。要求严格逐行一对一对应。

现在激活精确逐行翻译模拟器。该模拟器仅执行以下固定行为：
- 输入：多行纯文本。
- 输出：与输入行数完全相同的译文行。
- 每行输出仅包含对应原文的翻译结果。
- 禁止任何形式的额外内容，包括但不限于：
  - 编号、标记、引号、括号包裹
  - 前言、后记、解释、总结、统计信息
  - 空行（除非原文就是空行）
  - 合并、跳过或重复任何行
- 输出总行数必须精确等于 {len(texts)}。

模拟器立即处理以下输入文本（共 {len(texts)} 行）：

{texts_display}

模拟器输出开始："""
messages.append({"role": "user", "content": correction_msg})
```

- **延迟重试**：
```python
self.logger.info(f"[同一对话] 追加纠正指令（累计{len(messages)}条消息），继续重试...")
await asyncio.sleep(2)
continue
```

## 5. 重试次数对比表

| 重试场景 | 配置重试次数 | 实际重试次数 | 说明 |
|---------|---------|---------|------|
| 初始批量翻译 | `attempts`值 | 从日志看出为2次 | 默认-1（无限重试），日志中显示为 `批量翻译尝试 1/2` 和 `批量翻译尝试 2/2` |
| 批量拆分为单张 | 代码中为1次 | 日志中未观察到 | 代码中设置为固定1次，不受配置影响 |
| 安全限制纯文本回退 | 代码中为1次 | 日志中显示为2次 | 代码中设置为固定1次，但日志显示为 `最大尝试次数: 2` |
| 精确逐行翻译模拟器 | 单独1次 | 日志中未观察到 | 在同一对话中继续，不计入总尝试次数 |
| **HQ空内容降级** | 单图1次 + 文本2次 | - | 拆分单图重试1次，失败后文本翻译2次，最终失败返回原文 |

## 7. 重试机制的优缺点分析

### 7.1 优点
1. **渐进式降级**：从高质量批量翻译 → 高质量单张翻译 → 纯文本翻译，最大化成功率
2. **资源优化**：批量处理提高效率，仅在必要时降级到单张处理
3. **精确指令引导**：通过精确逐行翻译模拟器有效解决数量不匹配问题
4. **保持上下文**：同一对话重试保持上下文连贯性
5. **详细日志记录**：每个阶段都有清晰的日志记录，便于调试和问题追踪

### 7.2 缺点
1. **无限重试风险**：默认值-1可能导致无限循环（但有全局超时保护）
2. **固定单次尝试限制**：单张和纯文本回退固定为1次，不可配置
3. **批量大小影响**：批量大小影响重试策略，可能需要根据实际情况调整

## 9. 实际应用示例

### 9.1 常见错误及处理方式

#### 9.1.1 内容安全过滤错误
- **错误信息**：`'content_filter'` 或 `'safety'` 关键词
- **处理方式**：降级到单张处理，然后尝试纯文本翻译
- **日志示例**：
```
[HQ翻译] 批量翻译失败（安全限制），分割为 5 个单张批次重试...
[HQ翻译] 单张重试 2/5: 3条文本
[HQ翻译] 单张重试因安全限制失败，尝试纯文本翻译...
[HQ翻译] 纯文本翻译成功 2/5
```

### 9.1.2 翻译数量不匹配错误
- **错误信息**：`'mismatch'` 或 `'不匹配'` 关键词
- **处理方式**：降级到单张处理，或使用精确逐行翻译模拟器
- **日志示例**：
```
[Attempt 1/3] Translation count mismatch: expected 5, got 4. Retrying...
[同一对话] 追加纠正指令（累计4条消息），继续重试...
```

#### 9.1.3 HQ 空内容错误（新增）
- **错误信息**：`'returned empty content'`、`'api returned empty text'`、`'返回空内容'` 等
- **触发条件**：API 返回 HTTP 200 但 `finish_reason: stop` 且无实际内容
- **处理方式**：拆分为单图重试 → 失败后降级到文本翻译 → 失败后返回原文
- **与安全限制的区别**：空内容错误不会中断流程，会返回原文继续处理
- **日志示例**：

## 11. 常见问题解答

### 11.1 为什么默认重试次数是无限(-1)?
为了最大化翻译成功率，特别是在批量处理时。实际上，全局超时保护和用户取消机制可以防止真正的无限循环。

### 11.2 为什么单张重试和纯文本回退固定为1次?
为了避免在单张图片上浪费过多API调用，同时保持整体翻译进度。如果单张图片经过多次尝试仍然失败，通常表明该图片内容可能存在特殊问题。

### 11.3 精确逐行翻译模拟器如何工作?
它通过在同一对话中添加明确的指令，引导模型产生格式严格的输出。通过角色扮演("模拟器")和明确的规则，使模型更容易理解和执行精确的逐行翻译任务。

### 11.4 如何调整重试次数以平衡质量和效率?
- 对于质量优先：设置`attempts=5`或更高
- 对于速度优先：设置`attempts=2`或`attempts=3`
- 对于平衡模式：设置`attempts=3`并启用`enable_post_translation_check`

### 11.5 如何处理持续失败的特定图片?
如果某张图片在所有重试机制下都失败，可以考虑：
1. 手动编辑图片，移除可能触发内容过滤的元素
2. 使用不同的翻译模型或服务
3. 跳过该图片，继续处理其他图片

### 11.6 HQ 翻译返回空内容时系统怎么处理?
系统会自动触发三级降级策略：
1. **HQ 单图重试**：拆分批次为单图，逐张调用 HQ 翻译
2. **文本降级**：单图失败后降级到 openai 文本翻译（不发送图片）
3. **原文回退**：文本翻译也失败时，返回原文继续处理

与安全限制降级的关键区别：空内容失败不会中断整体流程，保证翻译任务可以继续进行。

### 11.7 重试时为什么显示"未匹配到作品"？
这通常是因为重试上下文中缺少 `image_name` 字段。已修复的版本会：
- HQ 批量翻译时，将第一张图片的 `image_name` 复制到合并上下文
- 单图重试时，从原始批次上下文中复制作品相关字段
- 确保术语加载时能正确识别作品名
