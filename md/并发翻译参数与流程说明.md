# 并发翻译参数与流程说明

## 核心参数定义

| 参数 | 默认值 | 说明 |
|------|--------|------|
| `batch_size` | 3 | 小批量大小，每次翻译请求包含的图片数 |
| `translation_concurrency` | 1 | 翻译并发数，同时发起的翻译请求数 |
| `context_size` | 0 | 上下文页数，翻译时携带前几页的译文/原文作为参考 |

## 批量大小概念

### 小批量（batch_size）
- 每次翻译 API 请求包含的图片数量
- 例如 `batch_size=3`：每次请求翻译 3 张图片的文本

### 大批量（big_batch_size）
- 大批量 = `batch_size × translation_concurrency`
- 例如 `batch_size=3, concurrency=4`：大批量 = 12 张图
- 用于术语延迟刷新策略

## 处理流程

### 1. 启动阶段
```
启动时：
├── current_concurrency = 1（默认并发数为1）
├── translation_semaphore = Semaphore(1)
├── 术语缓存策略 = 直接写入模式
└── concurrency_adjusted = True（标记为已调整状态）
```

### 2. 流水线并发处理
```
并发流水线：
├── 检测+OCR线程（顺序处理）
│   └── 逐张处理图片 → 放入翻译队列和修复队列
│
├── 翻译线程（批量处理）
│   ├── 收集 batch_size 张图片
│   ├── 使用 translation_semaphore 控制并发
│   └── 调用翻译 API
│
├── 修复线程（顺序处理）
│   ├── 从修复队列取图片
│   └── 执行修复
│
└── 渲染线程（等待翻译+修复都完成）
    └── 渲染并保存结果
```

### 3. 并发数恢复时机
```
修复线程停止时：
├── 设置 inpaint_worker_stopped = True
├── 恢复 current_concurrency = original_concurrency
├── 更新 translation_semaphore = Semaphore(新并发数)
├── 切换术语缓存策略为延迟写入模式
└── 输出日志：✅ [翻译] 修复线程已停止，恢复翻译并发数: 1 -> 4
```

## 术语缓存策略

### 策略切换规则
| 并发数 | 写入策略 | 说明 |
|--------|----------|------|
| = 1 | 直接写入 | 每次提取术语后立即写入文件 |
| > 1 | 延迟写入 | 缓存术语，每偶数个小批量完成后刷新 |

### 延迟刷新流程（串行发送时）
```
刷新策略：每完成偶数个小批量（2、4、6...）刷新一次术语

批次1完成 → 缓存术语
批次2完成 → 刷新术语到文件 ✅
批次3完成 → 缓存术语
批次4完成 → 刷新术语到文件 ✅
批次5完成 → 缓存术语
批次6完成 → 刷新术语到文件 ✅
...

翻译结束：
└── flush_all() 写入最后剩余的术语
```

## 上下文传递

### 上下文构建
```python
# context_size = 3 时
# 翻译当前页前，获取前3页的译文/原文作为上下文

上下文格式：
<|1|>第1页的句子
<|2|>第2页的句子
...
```

### 优先级
1. 优先使用译文（all_page_translations）
2. 无译文时使用原文（_original_page_texts）

## 修复队列与并发的关系

### 设计目的
1. **启动时并发=1**：确保前面翻译的术语能直接写入文件
2. **修复线程停止后恢复并发**：此时检测+OCR+修复都完成，可以用高并发加速剩余翻译
3. **术语延迟刷新**：高并发时缓存术语，下一轮并发开始时刷新，确保术语可被后续翻译使用

### 时序关系
```
时间轴：
├── 启动
│   ├── 检测+OCR 开始（顺序处理）
│   ├── 翻译 开始（并发=1）
│   └── 修复 开始（顺序处理）
│
├── 修复线程停止（所有图片检测+OCR+修复完成）
│   └── 恢复并发数: 1 → 4
│
├── 翻译继续（并发=4，加速剩余翻译）
│   └── 术语延迟刷新生效
│
└── 翻译完成
    └── flush_all() 写入最后的术语
```
