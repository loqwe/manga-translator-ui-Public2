# 广告过滤规则说明

## 概述

水印过滤功能用于自动过滤漫画中的广告、水印、版权声明等无关文本，避免将它们翻译并渲染到最终图片中。

**配置位置**: `examples/config.json`
```json
{
  "enable_watermark_filter": true
}
```

**规则文件**: `examples/config/watermark_filter.json`

---

## 过滤时机

**重要**: 过滤发生在 **翻译之前**，针对 **OCR 识别的原文**。

**流程**:
1. OCR 识别文本（韩文/日文/中文等）
2. 检查是否匹配过滤规则
3. 如果匹配 → 跳过该文本块，不翻译、不渲染
4. 如果不匹配 → 进入翻译流程

---

## OCR 误识别问题

### 问题场景

韩文/日文广告文本可能被 OCR 误识别为其他字符：

| 实际内容 | OCR 识别结果 | 翻译结果 |
|---------|------------|---------|
| `웹툰왕국뉴토끼7468` | `是7468` | 最快的漫画提供网站 Newtoki 468 |
| `품` | `品` | 歪头 |
| `최빠 만화` | `外可智是不人` | 最快的漫画提供网站 |

### 解决方案

除了添加原始韩文规则，还需要添加 **OCR 误识别后的文本** 到过滤规则：

```json
{
  "partial_match_patterns": [
    // 原始韩文
    "최빠 만화 제공 사이트",
    "만화왕국",
    "newtoki",
    
    // OCR 误识别结果（使用正则表达式）
    {
      "pattern": "^是\\d{3,5}$",      // 匹配 "是" + 3-5位数字
      "regex": true
    },
    {
      "pattern": "^品$",               // 单独的 "品" 字
      "regex": true
    },
    {
      "pattern": "外可智是不人",       // 误识别的文本片段
      "regex": false
    }
  ]
}
```

## 规则类型

### 1. 部分匹配 (partial_match_patterns)

只要文本**包含**规则中的任何一个模式，就会被过滤。

#### 字符串模式（不区分大小写）
```json
"newtoki"           // 匹配包含 "newtoki" 的任何文本
"최빠 만화 제공"     // 匹配包含这段韩文的任何文本
```

#### 正则表达式模式
```json
{
  "pattern": "(?i)\\bhttps?://[\\w.]+\\b",  // 匹配 http/https URL
  "regex": true,
  "case_sensitive": false
}
```

**正则表达式示例**:
```json
{
  "pattern": "^是\\d{3,5}$",          // 完全匹配 "是" + 3-5位数字
  "regex": true
}
```

### 2. 完全匹配 (exact_match_patterns)

文本必须**完全等于**规则（去除标点和空格后比较）。

```json
"exact_match_patterns": [
  "pfftt situ waras",
  "DIVASCANS.COM",
  "MANGA PARK"
]
```

## 正则表达式语法

### 基本语法
- `^` - 字符串开头
- `$` - 字符串结尾
- `.` - 任意字符
- `*` - 0次或多次
- `+` - 1次或多次
- `?` - 0次或1次
- `\d` - 数字 (0-9)
- `\w` - 字母、数字、下划线
- `\s` - 空白字符
- `[abc]` - 字符集合
- `[^abc]` - 排除字符集合
- `{n,m}` - 重复 n 到 m 次

### 标志
- `(?i)` - 不区分大小写
- `\b` - 单词边界

### 示例
```json
{
  "pattern": "^是\\d{3,5}$",              // 完全匹配: "是" + 3-5位数字
  "regex": true
}

{
  "pattern": "(?i)\\bhttps?://newtok",   // 匹配 HTTP/HTTPS + newtok
  "regex": true
}

{
  "pattern": "\\d{3,5}화",               // 3-5位数字 + "화"（韩文"集"）
  "regex": true
}
```

## 常见问题

### Q: 为什么添加了韩文规则还是没过滤？

**A**: 检查 OCR 识别结果。如果 OCR 把韩文误识别成了其他字符，需要添加误识别后的文本到规则中。

**解决方案**:
1. 查看日志中的 OCR 识别结果
2. 添加误识别的文本到过滤规则（建议使用正则）

### Q: 正常对话也被误过滤了怎么办？

**A**: 规则太宽泛。使用更精确的正则表达式或完全匹配模式。

**示例**:
```json
// ❌ 太宽泛
"品"

// ✅ 更精确
{
  "pattern": "^品$",          // 只匹配单独的 "品"
  "regex": true
}
```

### Q: 如何过滤所有 URL？

**A**: 使用正则表达式匹配 URL 模式：

```json
{
  "pattern": "(?i)\\bhttps?://[\\w.-]+\\.[a-z]{2,}\\b",
  "regex": true,
  "case_sensitive": false
}
```

### Q: 过滤规则何时生效？

**A**: 修改 `watermark_filter.json` 后，规则会在 **下次翻译任务开始时** 自动重新加载（缓存5分钟）。无需重启程序。
