# 翻译器重试机制详解

## 1. 重试次数设置

### 全局重试次数配置
- **默认值**：`attempts = -1`（无限重试）
- **配置来源**：从配置文件中读取，通过`parse_args`方法设置
- **继承关系**：`OpenAIHighQualityTranslator`继承自`CommonTranslator`，重试次数配置在父类中定义
- **同步机制**：设置`attempts`后会同步到`_max_total_attempts`
```python
# 同步 attempts 到 _max_total_attempts
self._max_total_attempts = self.attempts
```

### 重试次数的使用
- **主循环控制**：
```python
max_retries = self.attempts
attempt = 0
is_infinite = max_retries == -1
while is_infinite or attempt < max_retries:
    # 重试逻辑
```
- **全局尝试计数**：
```python
# 检查全局尝试次数
if not self._increment_global_attempt():
    raise Exception(f"达到最大尝试次数 ({self._max_total_attempts})，最后一次错误: {last_error_msg}")
```

## 2. 安全限制重试流程

### 批量翻译失败时的处理
1. **检测安全限制错误**：
```python
error_msg = str(last_batch_error).lower()
is_safety_error = 'content_filter' in error_msg or 'safety' in error_msg
```

2. **批量拆分为单张重试**：
```python
if (is_safety_error or is_mismatch_error) and len(batch_data) > 1:
    self.logger.warning(f"[HQ翻译] 批量翻译失败（{'安全限制' if is_safety_error else '数量不匹配'}），分割为单张批次重试...")
```

3. **单张重试次数设置**：
```python
# 单张重试（1次）
self._reset_global_attempt_count()
self._max_total_attempts = 1
```

4. **纯文本回退重试次数**：
```python
# 安全限制：尝试纯文本翻译（1次）
text_translator = OpenAITranslator()
text_translator.attempts = 1  # 纯文本只尝试1次
```

### 单张图片安全限制错误处理
1. **检测单张图片安全限制**：
```python
elif is_safety_error and len(batch_data) == 1:
    self.logger.warning(f"[HQ翻译] 单张图片因安全限制失败，尝试纯文本翻译...")
```

2. **纯文本回退重试次数**：
```python
text_translator = OpenAITranslator()
text_translator.attempts = 1
```

## 3. 数量不匹配重试流程

### 批量翻译数量不匹配处理
1. **检测数量不匹配错误**：
```python
is_mismatch_error = 'mismatch' in error_msg or '不匹配' in error_msg
```

2. **批量拆分为单张重试**：与安全限制共用相同的拆分逻辑，单张重试次数同样为1次

### 精确逐行翻译模拟器
1. **触发条件**：
```python
if len(translations) != len(texts):
    retry_attempt += 1
    retry_reason = f"Translation count mismatch: expected {len(texts)}, got {len(translations)}"
```

2. **重试次数控制**：
```python
if not is_infinite and attempt >= max_retries:
    raise Exception(f"Translation count mismatch after {max_retries} attempts: expected {len(texts)}, got {len(translations)}")
```

3. **同一对话重试机制**：
```python
# 使用同一对话继续重试：追加assistant回复和纠正指令
messages.append({"role": "assistant", "content": result_text})
messages.append({"role": "user", "content": correction_msg})
```

## 4. 重试次数对比表

| 重试场景 | 重试次数 | 说明 |
|---------|---------|------|
| 初始批量翻译 | `attempts`值 | 默认-1（无限重试） |
| 批量拆分为单张 | 1次 | 固定为1次，不受配置影响 |
| 安全限制纯文本回退 | 1次 | 固定为1次，不受配置影响 |
| 精确逐行翻译模拟器 | 计入`attempts` | 在同一对话中继续，计入总尝试次数 |

## 5. 重试流程总结

### 安全限制重试流程（3级降级）
1. **高质量批量翻译**：尝试次数 = `attempts`值（默认无限）
2. **高质量单张翻译**：尝试次数 = 1次
3. **纯文本翻译回退**：尝试次数 = 1次

### 数量不匹配重试流程（2种策略）
1. **批次拆分策略**：与安全限制共用相同的拆分逻辑，单张尝试1次
2. **精确逐行翻译模拟器**：在同一对话中继续，计入总尝试次数

## 6. 重试次数的影响

### 优点
1. **批量翻译多次尝试**：通过多次尝试提高批量翻译成功率
2. **单张翻译限制尝试**：避免在单张图片上浪费过多API调用
3. **纯文本回退单次尝试**：作为最后手段，仅尝试一次，失败则放弃

### 缺点
1. **无限重试风险**：默认值-1可能导致无限循环（但有全局超时保护）
2. **固定单次尝试限制**：单张和纯文本回退固定为1次，不可配置
3. **批量大小影响**：批量大小影响重试策略，可能需要根据实际情况调整

## 7. 建议优化方向

1. **可配置单张重试次数**：将单张重试次数也设为可配置项
2. **可配置纯文本回退次数**：允许用户配置纯文本回退的尝试次数
3. **渐进式批量调整**：不仅是拆分为单张，也可以先拆分为较小批次（如2-3张）
4. **自适应重试间隔**：根据错误类型动态调整重试间隔
5. **错误模式学习**：记录常见错误模式，预先采取措施避免
