# 术语分区存储说明

## 概述

术语现在按"是否识别到熟肉作品名"分为两个区域存放：

| 区域 | 存储位置 | 翻译时是否应用 |
|------|----------|----------------|
| **作品区** | `glossary.works[作品名]` | ✅ 会应用 |
| **未确定作品名区** | `glossary.unassigned` | ⚠️ 按条件应用（见下文） |

### 未确定作品名区的应用规则

`unassigned` 中的术语并非完全不应用，而是按以下规则：

1. **优先从 works 匹配**：翻译时先尝试从 `works` 中找到与当前图片路径对应的作品
2. **回退到 unassigned**：若 `works` 中找不到匹配作品，则从 `unassigned` 中筛选
   - 筛选条件：术语的 `raw_work_name` 字段 == 当前图片的生肉路径名
   - 只有 `raw_work_name` 匹配的术语才会被应用
3. **无匹配时返回空**：若都无法匹配，则不应用任何术语

**日志标识**：当使用 `unassigned` 中的术语时，日志会显示 `[unassigned:生肉名]`

## JSON 结构示例

```json
{
  "glossary": {
    "works": {
      "某作品A": {
        "Person": [
          {"original": "교주", "translation": "教主", "lang": "ko"}
        ],
        "Location": [],
        "Org": [],
        "Item": [],
        "Skill": [],
        "Creature": []
      },
      "某作品B": {
        "Person": [
          {"original": "主人公", "translation": "主角", "lang": "ja"}
        ],
        ...
      }
    },
    "unassigned": {
      "Person": [
        {"original": "未知角色", "translation": "某人", "lang": "auto"}
      ],
      "Location": [],
      "Org": [],
      "Item": [],
      "Skill": [],
      "Creature": []
    }
  }
}
```

说明：
- `glossary.works`：按熟肉作品名分桶的术语（翻译时会应用）
- `glossary.unassigned`：无法识别作品名的术语（按 `raw_work_name` 匹配后可应用）
- `lang` 字段：术语的语言代码（`ko`=韩语, `ja`=日语, `en`=拉丁字母语言(en/es/id), `auto`=其他）
- `raw_work_name` 字段：术语提取时的生肉路径名（用于 `unassigned` 匹配）
- 根级 `Person/Location/...`：**已弃用**，UI 不再支持旧格式

## 术语提取写回规则

当开启自动术语提取（`extract_glossary`）时：

1. **明确识别到作品名** → 写入 `glossary.works[作品名]`
   - 条件：推断的作品名 == 匹配到的作品名（name_mapping.json 映射成功）

2. **无法明确识别作品名** → 写入 `glossary.unassigned`
   - 包括：未配置 name_mapping、目录结构无法推断、回退到默认作品等情况

3. **混作品批次** → 禁用术语提取写回
   - 一个批次里混了多个作品时，不写回任何术语（避免污染）

## 术语管理对话框

### 左侧作品列表

```
📚 全部        ← 只读视图，显示所有作品区术语合并
🗃 无作品      ← 可编辑，对应 glossary.unassigned
某作品A (5)   ← 可编辑，对应 glossary.works["某作品A"]
某作品B (3)   ← 可编辑，对应 glossary.works["某作品B"]
```

### 右侧语言筛选条

- 语言选项来自 `dict/prompts/*.json`（例如 `ko.json / ja.json / en.json / es.json / id.json`）
- 下拉框默认包含：`全部语言` + 这些提示词语言

**语言筛选行为：**
- 选择特定语言时，只显示该语言的术语
- 新增/导入术语默认使用“当前筛选语言”；若当前为“全部语言”，则回退到自动检测
- 保存时保留所有语言字段（lang），不会丢失其他语言数据

### 术语列表操作

每行术语前新增复选框，支持批量操作：

```
[全选] [全不选] [反选]                    [转移选中→作品...]

[✓] SEONGHYEON  →  圣贤    [删除]
[✓] DAMI        →  多美    [删除]
[ ] LEE DAMI    →  李多美  [删除]
```

### 功能总结

- **查看/编辑**：选中"无作品"或具体作品可查看/编辑对应术语
- **语言筛选**：选择特定语言只显示该语言术语
- **单个转移**：右键菜单 → "转移术语到其他作品..."
- **批量转移**：勾选术语后点击"转移选中→作品..."
- **一键转移**：在"无作品"视图点击"无作品→作品..."按钮
- **保存时**自动维护根级兼容结构

## 翻译时的行为

1. **单作品翻译**：
   - 先尝试从 `works` 中找到匹配的作品术语
   - 若找不到，则从 `unassigned` 中按 `raw_work_name` 筛选匹配的术语
   - 若仍无匹配，返回空术语表

2. **混作品批次**：
   - 合并多个作品的术语（包括"默认"作品）
   - 不包含 `unassigned` 区的术语

3. **旧格式（无 works 结构）**：
   - 若 `glossary` 仅有根级分类（无 `works`），则剥离 `unassigned`
   - 此时 `unassigned` 中的术语不会被应用

## 迁移说明

### 旧格式处理

- **不再支持自动转换**：旧格式（仅根级分类，无 `works` 结构）不会被自动转换
- UI 会显示警告并拒绝加载，提示用户手动升级文件格式
- 这是为了避免误覆盖原文件

### "默认"作品迁移

- 若 `works` 中存在名为"默认"的作品，会自动迁移到"未确定作品名"区
- 迁移后"默认"作品会被删除

### 自动迁移规则

- `unassigned` 中带 `raw_work_name` 的条目，若能通过 `name_mapping.json` 映射到熟肉名，会自动迁移到对应作品
- 迁移在加载时自动执行并保存

### 手动转移

- 可在术语管理对话框中使用"无作品→作品..."按钮批量转移
- 支持勾选多个术语后点击"转移选中→作品..."
