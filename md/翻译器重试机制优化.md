# 翻译器重试机制优化

## 1. 安全限制重试流程

安全限制重试流程是针对OpenAI API返回的内容安全过滤错误设计的三级降级策略：

### 第一级：高质量批量翻译
- 首先尝试使用高质量翻译模式批量处理文本
- 利用上下文感知和格式保留功能提供高质量翻译

### 第二级：单张图片重试
- 当批量翻译因安全限制失败且批次中有多张图片时
- 将批次分割成单张图片单独处理
- 日志记录：`[HQ翻译] 批量翻译失败（安全限制），分割为 {n} 个单张批次重试...`

### 第三级：纯文本翻译回退
- 当单张图片仍因安全限制失败时
- 回退到纯文本翻译模式（不包含图片上下文信息）
- 日志记录：`[HQ翻译] 单张重试因安全限制失败，尝试纯文本翻译...`

### 核心代码
```python
# 批量失败时拆分
if (is_safety_error or is_mismatch_error) and len(batch_data) > 1:
    self.logger.warning(f"[HQ翻译] 批量翻译失败（{'安全限制' if is_safety_error else '数量不匹配'}），分割为 {len(batch_data)} 个单张批次重试...")

# 单张失败时回退到纯文本
if single_is_safety:
    self.logger.warning(f"[HQ翻译] 单张重试因安全限制失败，尝试纯文本翻译...")
```

## 2. 数量不匹配重试流程

数量不匹配重试流程针对翻译结果行数与原文行数不匹配的情况，包含两种处理策略：

### 批次拆分策略
- 当批量翻译因数量不匹配失败且批次中有多张图片时
- 将批次分割成单张图片单独处理
- 与安全限制重试共用相同的拆分逻辑

### 精确逐行翻译模拟器
- 在同一对话中继续，保持上下文连贯性
- 将错误的翻译结果作为assistant的回复添加到对话历史
- 添加精确的纠正指令作为user的新消息
- 通过角色扮演引导模型进入严格的翻译模式

### 精确逐行翻译模拟器指令
```python
correction_msg = f"""错误：先前输出包含 {len(translations)} 行译文，但原文共有 {len(texts)} 行。要求严格逐行一对一对应。

现在激活精确逐行翻译模拟器。该模拟器仅执行以下固定行为：
- 输入：多行纯文本。
- 输出：与输入行数完全相同的译文行。
- 每行输出仅包含对应原文的翻译结果。
- 禁止任何形式的额外内容，包括但不限于：
  - 编号、标记、引号、括号包裹
  - 前言、后记、解释、总结、统计信息
  - 空行（除非原文就是空行）
  - 合并、跳过或重复任何行
- 输出总行数必须精确等于 {len(texts)}。

模拟器立即处理以下输入文本（共 {len(texts)} 行）：

{texts_display}

模拟器输出开始："""
```

## 3. 重试机制的优势

### 安全限制重试流程优势
1. **渐进式降级**：从高质量批量翻译 → 高质量单张翻译 → 纯文本翻译
2. **最大化成功率**：即使部分内容触发安全限制，仍能完成大部分翻译
3. **资源优化**：批量处理提高效率，仅在必要时降级到单张处理

### 精确逐行翻译模拟器优势
1. **利用对话历史**：让模型能够看到自己之前的错误输出和明确的纠正指令
2. **角色扮演引导**：通过"模拟器"概念引导模型进入严格的翻译模式
3. **明确的约束条件**：详细列出输出必须遵循的规则，特别强调行数匹配
4. **保持上下文**：不会丢失之前的翻译内容，只是要求模型重新格式化输出
5. **提高成功率**：比简单重试更有效，因为明确指出了错误并提供修正方向

## 4. 未来优化方向

1. **配置化重试策略**：将重试次数和策略放入配置文件，方便调整
2. **增加统计信息**：记录各类错误的发生频率和成功率，帮助优化参数
3. **渐进式批量调整**：不仅是拆分为单张，也可以考虑先拆分为较小批次（如2-3张）再尝试
4. **自适应重试间隔**：根据错误类型和API响应动态调整重试间隔
5. **错误模式学习**：记录常见错误模式，预先采取措施避免
