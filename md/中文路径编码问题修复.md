# 中文路径编码问题修复

## 问题描述

在使用长图拼接功能时，当文件路径包含中文字符时，会出现以下错误：

```
[ WARN:36@382.101] global loadsave.cpp:275 cv::findDecoder imread_('D:\婕敾\鏈炕璇慭i m quitting erotic manga_Chapter\Chapter_01\024.jpg'): can't open/read file: check file path/integrity
[ WARN:38@382.101] global loadsave.cpp:275 cv::findDecoder imread_('D:\婕敾\鏈炕璇慭i m quitting erotic manga_Chapter\Chapter_01\025.jpg'): can't open/read file: check file path/integrity
[ WARN:37@382.101] global loadsave.cpp:275 cv::findDecoder imread_('D:\婕敾\鏈炕璇慭i m quitting erotic manga_Chapter\Chapter_01\026.jpg'): can't open/read file: check file path/integrity
[ WARN:39@382.101] global loadsave.cpp:275 cv::findDecoder imread_('D:\婕敾\鏈炕璇慭i m quitting erotic manga_Chapter\Chapter_01\023.jpg'): can't open/read file: check file path/integrity
```

这是因为 OpenCV 的 `imread` 函数在 Windows 系统上对中文路径的支持不完善，导致无法正确读取包含中文字符的图片文件。

## 原因分析

1. OpenCV 的 `imread` 和 `imwrite` 函数在处理文件路径时，对中文等非ASCII字符的支持不好
2. 在 Windows 系统上，这个问题尤为明显，因为 Windows 使用 UTF-16 编码存储文件名
3. 当路径中包含特殊中文字符时，编码转换可能会失败，导致 OpenCV 无法找到或读取文件

## 解决方案

将长图拼接功能中所有使用 OpenCV 读取图片的代码替换为使用 PIL (Python Imaging Library) 来读取和处理图片。PIL 对中文路径有良好的支持。

### 具体修改

1. 修改 `get_image_size` 函数，使用 PIL 替代 OpenCV 读取图片尺寸：

```python
# 修改前（使用 OpenCV）
def get_image_size(path: str) -> dict:
    try:
        img = cv2.imread(path, cv2.IMREAD_UNCHANGED)
        if img is not None:
            h, w = img.shape[:2]
            return {'path': path, 'width': w, 'height': h, 
                    'number': extract_number(path), 'filename': os.path.basename(path)}
    except Exception:
        pass
    return None

# 修改后（使用 PIL）
def get_image_size(path: str) -> dict:
    try:
        with Image.open(path) as img:
            w, h = img.size
            return {'path': path, 'width': w, 'height': h, 
                    'number': extract_number(path), 'filename': os.path.basename(path)}
    except Exception as e:
        self.progress.emit(f"[长图拼接] ⚠️ 无法读取图片: {os.path.basename(path)} - {e}")
        pass
    return None
```

2. 修改 `stitch_group` 函数，使用 PIL 替代 OpenCV 读取和处理图片：

```python
def stitch_group(args) -> tuple:
    batch_index, group = args
    try:
        # 读取图片
        imgs = []
        for info in group:
            try:
                with Image.open(info['path']) as img:
                    imgs.append((img.copy(), info))
            except Exception as e:
                self.progress.emit(f"[长图拼接] ⚠️ 无法读取图片: {info['filename']} - {e}")
        
        # ...省略部分代码...
        
        # 创建白色画布并拼接
        canvas = Image.new('RGB', (max_width, total_height), (255, 255, 255))
        y_offset = 0
        for img, info in imgs:
            # 居中放置
            x_offset = (max_width - img.width) // 2
            canvas.paste(img.convert('RGB'), (x_offset, y_offset))
            y_offset += img.height + self.margin
        
        # ...省略部分代码...
        
        # 保存为JPEG
        canvas.save(result_path, 'JPEG', quality=95, optimize=True)
```

## 优势

1. **更好的兼容性**：PIL 对各种操作系统上的中文路径都有良好支持
2. **更详细的错误信息**：添加了更详细的错误日志，便于排查问题
3. **代码简化**：PIL 的 API 更加简洁，代码可读性更好

## 注意事项

1. 此修复仅针对长图拼接功能，如果其他功能也有类似问题，可以采用同样的方法修复
2. PIL 和 OpenCV 在图像处理性能上有差异，但对于拼接功能来说，这种差异不会造成明显影响
