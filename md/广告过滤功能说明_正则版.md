# 广告过滤功能说明（正则表达式版）

## 功能概述

文本过滤功能支持三种匹配模式，可以自动跳过广告、水印等不需要翻译的文本区域。

匹配的文本区域会被完全跳过：
- ✅ 不翻译
- ✅ 不擦除
- ✅ 不渲染

## 配置文件位置

```
examples/filter_list.txt
```

## 三种过滤模式

### 1. 包含过滤（模糊匹配）

原文**包含**指定文本就过滤，不区分大小写。

**适用场景**：过滤包含特定关键词的文本

**示例**：
```
[包含过滤]
TEMPLESCAN
DISCORD.GG
```

**效果**：
- `TEMPLESCAN` ✅ 过滤
- `T.ME/TEMPLESCANS` ✅ 过滤（包含 TEMPLESCAN）
- `Visit TempleScanEsp.Net` ✅ 过滤（包含 TEMPLESCAN）

---

### 2. 精确过滤（完全匹配）

原文必须**完全等于**指定文本才过滤，不区分大小写。

**适用场景**：过滤短文本、单个字符、网址等

**示例**：
```
[精确过滤]
esp.net
WORLD
日
```

**效果**：
- `esp.net` ✅ 过滤
- `WORLD` ✅ 过滤
- `日` ✅ 过滤
- `NEW WORLD` ❌ 不过滤（不是完全匹配）

---

### 3. 正则过滤（正则表达式）

使用正则表达式匹配，不区分大小写。

**适用场景**：复杂模式、变体文本、多个相似词语

**示例**：
```
[正则过滤]
^>?STAFF$
CAP[IÍ]TULO\s*\d+
TRADUCCI[ÓO]N
REDRAW(ER)?
```

**正则语法说明**：

| 符号 | 含义 | 示例 | 匹配 |
|------|------|------|------|
| `^` | 行首 | `^STAFF$` | 只匹配完整的 "STAFF" |
| `$` | 行尾 | `^STAFF$` | 只匹配完整的 "STAFF" |
| `?` | 可选（0或1次） | `>?STAFF` | 匹配 "STAFF" 或 ">STAFF" |
| `\s` | 空格 | `CAP\s*\d+` | 匹配空格 |
| `\s*` | 0或多个空格 | `CAPÍTULO\s*9` | 匹配 "CAPÍTULO9" 或 "CAPÍTULO 9" |
| `\d` | 数字 | `\d+` | 匹配 1 个或多个数字 |
| `+` | 1次或多次 | `\d+` | 匹配 "9" 或 "123" |
| `[abc]` | 字符类 | `[ÓO]` | 匹配 Ó 或 O |
| `\|` | 或 | `A\|B` | 匹配 A 或 B |
| `()` | 分组 | `(ER)?` | 将 ER 作为整体可选 |

**效果**：
- `^>?STAFF$` → 匹配 "STAFF" 或 ">STAFF"（完整单词）
- `CAP[IÍ]TULO\s*\d+` → 匹配 "CAPÍTULO9"、"CAPITULO 10" 等
- `TRADUCCI[ÓO]N` → 匹配 "TRADUCCIÓN" 或 "TRADUCCION"
- `REDRAW(ER)?` → 匹配 "REDRAW" 或 "REDRAWER"
- `(CHINO|JAPONES)\s*-\s*ESPANOL` → 匹配 "CHINO-ESPANOL"、"JAPONES - ESPANOL" 等

## 使用建议

### 1. 优先使用正则过滤

正则表达式更灵活强大：
- ✅ 一条规则覆盖多种变体
- ✅ 支持边界匹配，避免误过滤
- ✅ 支持可选字符和重复

**示例**：
```
❌ 不好：添加 10 条包含过滤
   STAFF
   TRADUCTOR
   TRADUCTORES
   TRADUCCION
   TRADUCCIÓN
   ...

✅ 更好：1 条正则过滤
   TRADUCTOR(ES)?
   TRADUCCI[ÓO]N
```

### 2. 精确过滤适合短文本

短文本、单字符、网址等用精确过滤：
```
esp.net
日
©
```

### 3. 包含过滤适合长关键词

长的、唯一的关键词用包含过滤：
```
TEMPLESCAN
DISCORD.GG/ZBFYEDEKAD
```

### 4. 避免过度过滤

⚠️ 注意不要过滤正文内容：
```
❌ 不要：UNETE          # 会误过滤 "UNETE A NOSOTROS"（加入我们）
✅ 应该：^UNETE$        # 只过滤单独的 "UNETE"
```

## 编辑过滤列表

### 1. 打开文件
```
examples/filter_list.txt
```

### 2. 添加规则
在对应区域添加规则，每行一条：
```
[包含过滤]
YOUR_KEYWORD

[精确过滤]
YOUR_EXACT_TEXT

[正则过滤]
YOUR_REGEX_PATTERN
```

### 3. 测试规则
直接翻译一张图片，查看日志输出：
```bash
python -m manga_translator --mode batch --target-lang CHS --translator gpt4o-hq <图片路径>
```

查找日志中是否有如下输出：
```
过滤文本区域 (正则匹配): "STAFF" -> 匹配: "^>?STAFF$"
过滤文本区域 (包含匹配): "TEMPLESCAN" -> 匹配: "templescan"
```

### 4. 重新加载
修改后重启应用，或调用 `load_filter_list(force_reload=True)`

## 常用正则模式

### 章节号
```
(CHAPTER|CAP[IÍ]TULO|第)\s*\d+\s*(章|话)?
```

### 职位/角色
```
(STAFF|EDITOR|TRANSLATOR|REDRAW(ER)?|TYPER)
```

### 网址
```
(https?://)?[\w.-]+\.(com|net|org|me)(/\S*)?
```

### 社交媒体
```
(DISCORD\.GG|T\.ME|TWITTER\.COM|INSTAGRAM\.COM)
```

### 语言对
```
(中文|日[本语]|韩[国语]|英[文语])\s*[-→]\s*(中文|简体|繁体)
```

---

## 注意事项

1. **正则不区分大小写**：所有正则都带 `re.IGNORECASE` 标志
2. **特殊字符需转义**：`.` `?` `*` `+` `|` `(` `)` `[` `]` `^` `$` 等需要转义
3. **测试很重要**：添加规则后务必测试，避免误过滤正文
4. **注释用 #**：以 `#` 开头的行会被忽略
5. **UTF-8 编码**：文件必须使用 UTF-8 编码保存

## 性能优化

### 规则数量
- ✅ 包含过滤：~10 条
- ✅ 精确过滤：~5 条
- ✅ 正则过滤：~20 条

### 匹配顺序
1. 先检查精确匹配（最快）
2. 再检查包含匹配（较快）
3. 最后检查正则匹配（较慢）

### 优化建议
- 能用精确过滤就不用包含过滤
- 能用包含过滤就不用正则过滤
- 正则尽量简洁，避免复杂的回溯
